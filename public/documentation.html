WHAT THE APP CAN DO:

The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

URL ENDPOINT AND THEIR METHOD:

Business Logic URL Description HTTP Method Query Parameters Request body data format Response body data format
Get All Movies /movies Returns a list of all movies GET None None JSON of movie array
Get Movies by Movie Title /movies/:title Returns a movie by its specified title GET title (URL parameter) None JSON obejct with movie
Get Movies by Genre /movies/Genre/:GenreName Returns a movie by its genre GET name (URL parameter) None JSON object with genre name
Get Movies by Director's Name /movies/director/:directorsName Returns movies by the directors name GET name (URL parameter) None JSON object about director and movie they directed
Gets all existing users /users Displays all existing users GET username (URL parameter) None JSON object with information
Create a New User /users Allows a new user to create profile POST username (URL parameter) None JSON object with information
Update a User Name /users/:username Allows a user to update name and other info PUT Username (URL parameter) JSON object with updated user info Confirmation message
Add a Favorite Movie /users/:username/movieid Allows a user to add a favorite movie POST username (URL parameter) JSON object with movie title Updated list of favorite movies
Remove a Movie From Favorites /users/:username/movieid Allows a user to remove movies from the favorite list DELETE username, movieid None Updated list of favorite movies or confirmation message
Delete a User /users/:username Allows a user to be deleted DELETE username (URL parameter) None Confirmation or error message
